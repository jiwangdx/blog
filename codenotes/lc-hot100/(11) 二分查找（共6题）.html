11 二分查找

[中等]

给定一个排序数组和一个目标值，在数组中找到目标值，并返回其索引。请必须使用时间复杂度为 O(log n) 的算法。

class Solution {
    public int searchInsert(int[] nums, int target) {
        // 左闭右闭
        int start = 0;
        int end = nums.length-1;

        while(start<=end){
            int mid = start+(end-start)/2;

            if(nums[mid]==target){
                return mid;
            }
            else if(nums[mid]>target){
                end = mid-1;
            }
            else{
                start = mid+1;
            }
        }
        return start;
    }
}

[中等]

给你一个满足下述两条属性的 m x n 整数矩阵：

  • 每行中的整数从左到右按非严格递增顺序排列。
  • 每行的第一个整数大于前一行的最后一个整数。

给你一个整数 target ，如果 target 在矩阵中，返回 true ；否则，返回 false 。

**思路：**设m为矩阵的行数，n为矩阵的列数

首先观察题目中的矩阵，若是将矩阵行和列都完全展开，都放到一维数组中

1，3，5，7，10，11，16，20，23，30，34，60 即是非递减的，此时对一维数组进行二分查找就很容易了

能不能不使用额外的数组？ 此时就涉及到 元素在 "二维数组" 和 "一维数组" 上的位置的映射关系

如果一个元素在展开后的一维数组中的位置为 i，那么它在二维数组中的位置会是（i/n, i%n）

  • 例如 3 ，在一维数组中位置 i = 1 , 在二维数组中位置是（1/4，1%4）== （0,1）
  • 例如16，在一维数组中位置 i = 6 , 在二维数组中位置是（6/4，6%4）== （1,2）
  • 例如30，在一维数组中位置 i = 9 , 在二维数组中位置是（9/4，9%4）== （2,1）

现在知道了映射关系，我们可以直接**假设 **已经将二维数组展开为一维数组了，对虚拟的一维数组进行二分查找

class Solution {
    public boolean searchMatrix(int[][] matrix, int target) {
        int m = matrix.length; // 行数
        int n = matrix[0].length; // 列数

        // 假设已经展开为一维数组
        int left = 0; // 二分查找边界
        int right = m * n - 1;

        while (left <= right) {
            // 此时，此处的mid是展开后的一维数组的下标
            int mid = (left + right) >>> 1;
            // 需要将mid映射到二维数组的位置,根据映射关系（i/n, i%n）
            int num = matrix[mid / n][mid % n];
            
            if (num == target) {
                return true;
            }
            if (num < target) {
                left = mid + 1;
            } else {
                right = mid - 1;
            }
        }
        return false;
    }
}

[中等]

给你一个按照非递减顺序排列的整数数组 nums，和一个目标值 target。请你找出给定目标值在数组中的。

如果数组中不存在目标值 target，返回 [-1, -1]。

你必须设计并实现时间复杂度为 O(log n) 的算法解决此问题。

**思路：**对于非递减排序的整数数组，且要求时间复杂度为 O(log n)，那么一定会想到二分查找

如何找到目标值的？

  • 假设我们现在要找目标值的左边界，其实就是在当二分查找的过程中，当 num[mid] == target 的时候

我们不要立马返回，因为当前mid下标的元素的左边可能还有等于target的元素，即我们还应该往左边去找一找

即 此时只需要让 right = mid -1 继续循环去找即可

  • 反之，假设现在要找目标值的右边界，其实就是在当二分查找的过程中，当 num[mid] == target 的时候

我们不要立马返回，因为当前mid下标的元素的右边可能还有等于target的元素，即我们还应该往右边去找一找

即 此时只需要让 left = mid + 1 继续循环去找即可

  • <font style="color:rgb(15, 17, 21);background-color:rgb(235, 238, 242);">right</font><font style="color:rgb(15, 17, 21);background-color:rgb(235, 238, 242);">right</font>
class Solution {
    public int[] searchRange(int[] nums, int target) {
        int first = firstSolution(nums, target);
        int end = endSolution(nums, target);
        return new int[] { first, end };
    }

    public int firstSolution(int[] nums, int target) {
        int left = 0;
        int right = nums.length-1;

        while (left <= right) {
            int mid = left + (right - left) / 2;

            if (nums[mid] < target) {
                left = mid + 1;
            } else if (nums[mid] > target) {
                right = mid - 1;
            } else {
                // 继续往左边找一找
                right = mid - 1;
            }
        }
        // 此时left 指向的是第一个大于等于 target的位置
        if (left < 0 || left >= nums.length)
            return -1;
        if (nums[left] != target)
            return -1;

        return left;
    }

    public int endSolution(int[] nums, int target) {
        int left = 0;
        int right = nums.length-1;

        while (left <= right) {
            int mid = left + (right - left) / 2;

            if (nums[mid] < target) {
                left = mid + 1;
            } else if (nums[mid] > target) {
                right = mid - 1;
            } else {
                left = mid + 1;
            }
        }
         // 此时right 指向的是最后一个小于等于 target的位置
        if (right < 0 || right >= nums.length)
            return -1;
        if (nums[right] != target)
            return -1;

        return right;
    }

}

已知一个长度为 n 的数组，预先按照升序排列，经由 1 到 n 次 旋转 后，得到输入数组。例如，原数组 nums = [0,1,2,4,5,6,7] 在变化后可能得到：

  • 若旋转 4 次，则可以得到 [4,5,6,7,0,1,2]
  • 若旋转 7 次，则可以得到 [0,1,2,4,5,6,7]

注意，数组 [a[0], a[1], a[2], ..., a[n-1]]旋转一次 的结果为数组 [a[n-1], a[0], a[1], a[2], ..., a[n-2]] 。

给你一个元素值 互不相同 的数组 nums ，它原来是一个升序排列的数组，并按上述情形进行了多次旋转。请你找出并返回数组中的 最小元素 。

你必须设计一个时间复杂度为 O(log n) 的算法解决此问题。

**思路：**对于非递减排序的整数数组，且要求时间复杂度为 O(log n)，那么一定会想到二分查找

观察题目，虽然经过了旋转，但是也是 有两段非递减的升序段，例如【3，4，5】、【1、2】

这时候二分，mid 在哪一段升序段是不固定的。我们观察一下，假设是这样

3 4 5 1 2 
L m     R
如果 nums[mid]  > nums[R] 那么mid一定是在第一段升序段的，此时应该让 L = mid +1

5 1 2 
L m R
如果 nums[mid]  < nums[R] 那么mid一定是在第二段升序段的，此时应该让 R = mid

但是如果是 nums[mid] == nums[R] 呢，其实此时就无法区分出mid在第一段还是第二段升序段了，例如

2 2 2 1 2 nums[mid] == nums[R] mid在第一段

1 0 1 1 1 1 nums[mid] == nums[R] mid在第二段

因此此时只让R--或者L++即可

最终L和R一定会相逢，最后返回nums[L]即可

但题目里其实说明了，，所以不会存在nums[mid] == nums[R]的情况。

class Solution {
    public int findMin(int[] nums) {
        int left = 0, right = nums.length - 1;
        /**
            循环结束时 left == right
            此时区间缩小到一个元素，这个元素就是最小值
         */

        while(left < right){
            int mid = (left+right) >>> 1;
            if(nums[mid] > nums[right]){  // mid 在第一段升序段
                left = mid + 1;
            }else if(nums[mid] < nums[right]){ // mid 在第二段升序段
                right = mid;
            }
            /*else{  // 无法确定,只让R--或者L++即可
                //但题目里其实说明了，元素值互不相同，所以不会存在nums[mid] == nums[R]的情况
                right--;
            }*/
        }
        return nums[left];
    }
}

[中等]

整数数组 nums 按升序排列，数组中的值 互不相同 。

在传递给函数之前，nums 在预先未知的某个下标 k（0 <= k < nums.length）上进行了 旋转，使数组变为 [nums[k], nums[k+1], ..., nums[n-1], nums[0], nums[1], ..., nums[k-1]]（下标 从 0 开始 计数）。例如， [0,1,2,4,5,6,7] 在下标 3 处经旋转后可能变为 [4,5,6,7,0,1,2] 。

给你 旋转后 的数组 nums 和一个整数 target ，如果 nums 中存在这个目标值 target ，则返回它的下标，否则返回 -1 。

你必须设计一个时间复杂度为 O(log n) 的算法解决此问题。

**思路：**先找到旋转数组中的最小值所在的下标索引 minIndex

此时两个升序段就被区分出来了：[0, minIndex-1]、[minIdex, nums.lenght-1]

此时拿 target 和 nums[right] 进行比较，就能知道target在哪一段

  • nums[right] < target，则 target 在第一升序段，在其内进行二分查找
  • nums[right] > target，则 target 在第二升序段，在其内进行二分查找
  • nums[right] == target 则直接返回 right
class Solution {
    // 先找到最小值，区分出两段后，在其中一段再按二分法查找target值
    public int search(int[] nums, int target) {
        // 先找到最小值所在的下标索引,此时两个升序段就被区分出来了,[0, minIndex-1]、[minIdex, nums.lenght-1]
        int minIndex = findMin(nums);
        
        // nums[right]指向的元素是更小的那一段的最大元素（最后一个元素）
        // 此时拿target和nums[right]进行比较，就能知道target在哪一段
        int left = 0, right = nums.length - 1;
        if (nums[right] < target) {
            // target 在第一升序段,在其内进行二分查找
            return searchBinary(nums, target, 0, minIndex - 1);
        } else if (nums[right] > target) {
            // target 在第二升序段,在其内进行二分查找
            return searchBinary(nums, target, minIndex, right);
        } 
        else{
            return right;
        }
    }

    // 二分法
    public int searchBinary(int[] nums, int target, int left, int right) {
        while (left <= right) {
            int mid = (left + right) >>> 1;
            if (nums[mid] < target) {
                left = mid + 1;
            } else if (nums[mid] > target) {
                right = mid - 1;
            } else {
                return mid;
            }
        }
        return -1;
    }

    // 找出最小值的所在索引，参考上一题153.寻找旋转排序数组中的最小值的答案
    public int findMin(int[] nums) {
        int left = 0, right = nums.length - 1;
        while(left < right){
            int mid = (left + right) >>> 1;
            if(nums[mid] > nums[right]){  // mid 在第一段升序段
                left = mid + 1;
            }else if(nums[mid] < nums[right]){ // mid 在第二段升序段
                right = mid;
            }else{  // 无法确定,只让R--或者L++即可
                left++;
            }
        }
        return left;
    }
}

[困难]

给定两个大小分别为 m 和 n 的正序（从小到大）数组 nums1 和 nums2。请你找出并返回这两个正序数组的 中位数 。

算法的时间复杂度应该为 O(log (m+n)) 。

思路：

举个例子, [1,2,5,6] 和 [3,4,7,8,9] 中位数答案是5，抽象的想象有一根长度固定的绳子把 [3,4] 和 [1,2,5] 串起来了

此时绳子中最大的数(5)就是答案中位数(5)，这是规定。我们可以设定两个分割点，i = 3(下标)，j = 2(下标)

i 左边的数(1,2,5) 和 j 左边的数(3,4) 就是在绳子上的数

要想达到这样的规定，要满足一定的条件

  • nums1[i-1] <= nums2[j] 即 5<=7
  • nums2[j-1] <= nums2[i] 即 4<=6

当我们找到符合上述两个条件的i和j时

  • 如果总个数是奇数，就返回绳子上最大的数
  • 如果总个数是偶数，就返回绳子上最大的数和非绳子上最小的数的平均值

而i和j在一条绳子上,且绳子的长度是固定的， i+j = (m+n+1) /2，因此一旦i确定了，则j也确定了

要确定nums1上的分割点i，使用二分查找

class Solution {
    public double findMedianSortedArrays(int[] nums1, int[] nums2) {
        if(nums1.length>nums2.length) return findMedianSortedArrays(nums2,nums1);

        int m = nums1.length;
        int n = nums2.length;

        int left = 0;
        int right = m;
        // 虚拟分割点i j 分别表示拿出多少个元素分到左半部分
        while(left<=right){
            int i = (left+right) >>> 1;

            int j = (m+n+1)/2 - i;

            int left1 = (i==0) ? Integer.MIN_VALUE : nums1[i-1];
            int right1 = (i==m) ? Integer.MAX_VALUE : nums1[i];
            int left2 = (j==0) ? Integer.MIN_VALUE : nums2[j-1];
            int right2 = (j==n) ? Integer.MAX_VALUE : nums2[j];
// 看看是否满足要求
            if(left1<=right2 && left2 <= right1){
                // 满足要求
                if((m+n)%2==0){
                    return (Math.max(left1,left2)+Math.min(right1,right2))/2.0;
                }
                return Math.max(left1,left2);
            }
            else if(left1>right2){
                right = i-1;
            }
            else if(left2>right1){
                left = i+1;
            }
        }

        return 0;
            
    }
}
