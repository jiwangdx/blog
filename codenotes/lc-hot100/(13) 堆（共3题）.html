13 堆

jiwang3/4/26About 5 min代码笔记LC Hot100堆

背景知识：

  • 堆的逻辑结构和完全二叉树一致，但底层实现是数组（完全二叉树的层序遍历结果存储在数组）
  • 堆每次弹出元素，是从堆顶弹出
  • 堆又名优先队列，例如 Java 中的
  • 大顶堆：

[中等]

给定整数数组 nums 和整数 k，请返回数组中第 **k** 大的元素。

请注意，你需要找的是数组排序后的第 k 个最大的元素，而不是第 k 个不同的元素。

你必须设计并实现时间复杂度为 O(n) 的算法解决此问题。

思路：

如果题目要求找最大，那么就用小顶堆。如果题目要求找最小，那么就用大顶堆

class Solution {
    public int findKthLargest(int[] nums, int k) {
        // 找到数组第k个最大的元素
        // 3 2 1 5 6 4    k=2
        // 作一个小顶堆，只维护k个元素,这样写就是小顶堆
        Queue<Integer> queue = new PriorityQueue<>();
        // new PriorityQueue<>((a, b) -> (b - a)); 如果这样写就是大顶堆
        
        for (int i = 0; i < nums.length; i++) {
            queue.offer(nums[i]);
            if(queue.size() > k){
                queue.poll();
            }
        }
        
        return queue.peek();
    }
}

[中等]

给你一个整数数组 nums 和一个整数 k ，请你返回其中出现频率前 k 高的元素。你可以按 任意顺序 返回答案。

**思路：**同上，题目要求前k高，那么就是使用小顶堆，并且堆内只维持k个元素。

与前题不同之处，在于此题需要存储的是"频率"，但本质是相同的。

  • 我们需要先统计出所有元素的"频率"，存储到Map中，key是元素值，value是元素的"频率"
  • 构建小顶堆，堆中存储的是 Map的<key,value> 键值对，并且排序规则是以value为比较, 即以元素的"频率"大小为比较依据，这样就能使得 "频率" 小的键值对 会被放在堆顶
  • 遍历Map中的所有键值对
    • 先把键值对加入堆中
    • 再判断堆中元素的数量是否超过k个，若超过，则弹出堆顶的元素，这就维持了堆中元素是k个，且注意此时弹出的是频率最小的键值对，那么堆内留下的就是频率偏大的键值对
    • 因为Map中所有的键值对都会加入到堆中，并且经过调整和弹出，会把所有频率小的值弹出
    • 最终遍历完成后，堆中留下的就是k个频率最大的键值对
  • 将堆中剩余的k个键值对都取出其key放到数组中返回即可
class Solution {
    public int[] topKFrequent(int[] nums, int k) {
        // 前k个高频元素,小顶堆
        int[] res = new int[k];
        
        // 统计每个元素的"频率"，存储到Map中，key是元素值，value是元素的"频率"
        Map<Integer,Integer> map = new HashMap<>();
        for(int num : nums){
            map.put(num, map.getOrDefault(num, 0) + 1);
        }

        // 使用小顶堆，堆中存储的是 Map的<key,value> 键值对
        // 并且排序规则是以value为比较, 即以元素的"频率"大小为比较
        Queue<Map.Entry<Integer,Integer>> queue = new PriorityQueue<>(
            (o1,o2)-> o1.getValue()-o2.getValue()  // 注意这样写是小顶堆，反过来就是大顶堆
        );
        for(Map.Entry<Integer,Integer> entry : map.entrySet()){
            queue.add(entry);
            // 先加进去，堆会帮助我们自动排序
            if(queue.size() > k){
                queue.poll();
            }
        }
        
        // 将堆内剩余k个元素结果放到数组中返回。剩余k个元素即"频率"最高的k个元素
        for(int i = 0; i < k; i++){
            res[i] = queue.poll().getKey(); // 由于堆中存的是键值对，所以这里需要getKey
        }
        return res;
    }
}

[困难]

题目：中位数是的中间值。如果列表的大小是偶数，中位数是两个中间值的平均值。

  • 例如 arr = [2,3,4] 的中位数是 3 。
  • 例如 arr = [2,3] 的中位数是 (2 + 3) / 2 = 2.5 。

实现 MedianFinder 类:

  • MedianFinder() 初始化 MedianFinder 对象。
  • void addNum(int num) 将数据流中的整数 num 添加到数据结构中。
  • double findMedian() 返回到目前为止所有元素的中位数。与实际答案相差 10(-5) 以内的答案将被接受。

思路：

人为构造一个大顶堆 leftHalf 和 一个小顶堆 rightHalf，且需遵守如下规定

  • 当数据流元素数量为偶数：规定 leftHalf 和 rightHalf 大小相同，此时动态中位数为两者堆顶元素的平均值
  • 当数据流元素数量为奇数：规定 leftHalf 比 rightHalf ，此时动态中位数为 leftHalf 的堆顶数
  • 规定 
class MedianFinder {
    private final PriorityQueue<Integer> left = new PriorityQueue<>((a, b) -> b - a); // 大根堆
    private final PriorityQueue<Integer> right = new PriorityQueue<>(); // 小根堆

    public void addNum(int num) {
        if (left.size() == right.size()) {
            // 这时候要把合适的值放在左边，这个合适的值一定 大于左边的所有 && 小于右边的所有
            // 如果直接插在左边，并不能保证小于右边的所有
            // 所以要先插右边，通过小根堆搞出来右边最小的，再把这个数拿到左边
            right.offer(num);
            left.offer(right.poll());
        } else {
            // 这时候要放在右边，同样，也要保证这个数要守规矩，大于左边，所以要先求左边的最大
            left.offer(num);
            right.offer(left.poll());
        }
    }

    public double findMedian() {
        if (left.size() > right.size()) {
            return left.peek();
        }
        return (left.peek() + right.peek()) / 2.0;
    }
}
