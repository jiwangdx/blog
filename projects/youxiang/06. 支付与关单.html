06. 支付与关单

jiwang3/24/26About 2 min项目《优享生活圈》支付关单

一、未支付订单到期自动关闭

使用 Spring Task 定时任务，定期扫描超时未支付的订单，将其状态更新为“已关闭”。

@Component
public class OrderCloseTask {
    
    @Scheduled(cron = "0 * * * * ?")  // 每分钟执行一次
    public void closeExpiredOrders() {
        // 1. 查询创建超过15分钟且状态为“待支付”的订单
        List<Order> expiredOrders = orderService.queryExpiredOrders(15);
        
        // 2. 逐个关闭订单
        for (Order order : expiredOrders) {
            orderService.closeOrder(order.getId());
        }
    }
}

关键点：

  • @Scheduled 注解定义执行频率
  • 需要在启动类加 @EnableScheduling 开启定时任务

二、支付和关单的并发处理

2.1 问题场景

用户支付和定时关单可能同时操作同一笔订单，导致：

  • 用户付了钱，订单却被关单
  • 或者关单后用户又支付成功

2.2 解决方案：乐观锁

订单表增加 version 字段：

ALTER TABLE `order` ADD COLUMN `version` int DEFAULT 0;

更新时带上版本号条件：

UPDATE `order` 
SET status = #{newStatus}, version = version + 1
WHERE id = #{orderId} 
  AND status = #{oldStatus}
  AND version = #{version}

代码逻辑：

// 支付方法
public boolean payOrder(Long orderId, Integer version) {
    // 尝试把订单从“待支付”改成“已支付”
    // 只有订单确实是“待支付”且版本号没被人改过，才能成功
    int updated = updateStatus(orderId, "PAID", "WAIT_PAY", version);
    return updated > 0;  // 成功返回true
}

// 关单方法
public boolean closeOrder(Long orderId, Integer version) {
    // 尝试把订单从“待支付”改成“已关闭”
    // 只有订单确实是“待支付”且版本号没被人改过，才能成功
    int updated = updateStatus(orderId, "CLOSED", "WAIT_PAY", version);
    return updated > 0;  // 成功返回true
}

2.3 并发执行效果

场景支付线程关单线程结果
同时操作更新成功（version+1）更新失败（version不匹配）只支付成功，关单失败
同时操作更新失败更新成功只关单成功，支付失败

无论谁先成功，另一个操作必然失败，保证订单状态最终一致，不会出现既支付又关单的混乱。
